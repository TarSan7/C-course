Definition and description of functions


If variables and constants store the state of the program, then the functions determine its actions.

A formal definition of a function is as follows:

1

2

3

4		 type function_name (parameters)

{

    executable_expressions

}

The first line represents the title or signature of the function. The return type of the function is indicated first. If the function does not return any value, then the void type is used.

Then comes the name of the function, which represents an arbitrary identifier, to which the same rules apply as for naming variables.

After the function name in brackets there is an enumeration of parameters. The function may not have parameters, in this case empty brackets are indicated.

After the function header in curly brackets is the body of the function, which contains the executable expressions.

To return the result, the function uses the return statement, which has the following uses:

1

2		 return ;

return expression;

The first form can be used in those functions that do not return any value, that is, they have void as the return type.

The second form applies if the function has any type except void as the return type. The expression after the return statement represents the return value.

To perform a function, it must be called. The function call is carried out in the form:

function_name (arguments);

After the name of the function, brackets are indicated in which the arguments are listed - values ​​for the parameters of the function.

The hello function is defined here, which simply prints the string "Hello!" To the console. This function is of type void, that is, in fact, it returns nothing.

Any program must have at least one function that has a fixed name main (). It is with the main function that execution begins. Therefore, if we want to execute the hello function, then we need to call it in the main function.

It is also worth noting that the main function has the type int as the return type, so it must necessarily return some value of the int type using the return statement. Therefore, at the end of the body of the function is a call to return 0 ;. According to conventions, as an indicator of successful execution, the main function returns the number 0.

Function prototype or description

Here it is worth considering an important point - the compiler must know about the function before calling it. Therefore, a function call must occur after its definition, as in the case above. In some languages ​​this does not matter, but in the C language it plays a big role. And if, for example, we first call, and then define a function, then we will get an error at the compilation stage, as in the following case:
