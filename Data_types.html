Data types


A variable has a specific type. And this type determines what values ​​a variable can have and how many bytes in memory it will occupy. The following basic data types are defined in C:

  • char: represents a single character. It occupies 1 byte (8 bits) in memory. Can store any value from the range -128 to 127

  • unsigned char: represents a single character. It occupies 1 byte (8 bits) in memory. Can store any value from a range from 0 to 255

  • signed char: same as char

  • short int: represents an integer ranging from –32768 to 32767. It occupies 2 bytes (16 bits) in memory.

  • short: same as short int.

  • unsigned short int: represents an integer in the range from 0 to 65535. It occupies 2 bytes (16 bits) in memory.

  • unsigned short: same as unsigned short int.

  • signed short int: same as short int

  • signed short: same as short int.

  • int: represents an integer. Depending on the processor architecture, it can take up 2 bytes (16 bits) or 4 bytes (32 bits). The limit value range, respectively, can also vary from –32768 to 32767 (with 2 bytes) or from −2 147 483 648 to 2 147 483 647 (with 4 bytes).

  • unsigned int: represents a positive integer. Depending on the processor architecture, it can occupy 2 bytes (16 bits) or 4 bytes (32 bits), and because of this, the range of limit values ​​can vary: from 0 to 65535 (for 2 bytes), or from 0 to 4,294,967,295 (for 4 bytes).

  • unsigned: same as unsigned int

  • signed int: same as int

  • signed: same as int

  • long int: represents an integer ranging from −2,147,483,648 to 2,147,483,647. It occupies 4 bytes (32 bits) in memory.

  • long: same as long int.

  • signed long int: same as long int.

  • signed long: same as long int.

  • unsigned long int: represents an integer in the range 0 to 4,294,967,295. It occupies 4 bytes (32 bits) in memory.

  • unsigned long: same as unsigned long int.

  • long long int: represents an integer ranging from −9 223 372 036 854 775 808 to +9 223 372 036 854 775 807. Usually, it occupies 8 bytes (64 bits) in memory.

  • long long: the same as long long int.

  • signed long long int: same as long long int

  • signed long long: same as long long int.

  • unsigned long long int: represents an integer in the range from 0 to 18 446 744 073 709 551 615. Usually it occupies 8 bytes (64 bits) in memory.

  • unsigned long long: the same as unsigned long long int.

  • float: represents a floating point ordinary number in the range +/- 3.4E-38 to 3.4E + 38. 4 bytes (32 bits) in memory

  • double: represents a double-precision floating-point number in the range +/- 1.7E-308 to 1.7E + 308. It occupies 8 bytes (64 bits) in memory

  • long double: represents a double precision floating-point number in the range +/- 3.4E-4932 to 1.1E + 4932. It occupies 10 bytes (80 bits) in memory. On some systems, it can occupy 96 and 128 bits.

  • void: type without value

If we need to define several variables of the same type, then we can indicate their names after the data type, separated by commas: int a, b

A variable of type char takes one character in single quotes as a value: char c = 'd'. You can also assign a number from the range specified above in the list: char c = 120. In this case, the value of the variable c will be the character that has the code 120 in the ASCII character table.

In the above list, for each type, the size it occupies in memory is indicated. However, it is worth noting that compiler developers can choose the maximum size limits for types based on the hardware capabilities of the computer. The standard sets only the minimum values ​​that should be. For example, for int and short types, the minimum value is 16 bits, for long types, 32 bits. In this case, the size of the long type must be no less than the size of the int type, and the size of the int type must be no less than the size of the short type. But in general, the types use the sizes indicated above when describing data types.

However, there are situations when you need to know exactly the size of a certain type. And for this, C has a sizeof () operator, which returns the size of the memory in bytes that the variable occupies:
