Basic concepts

Type conversion


If values ​​of different types are involved in arithmetic operations, then the compiler implicitly tries to cast them to the same type. In addition, when we assign a value to a variable, this value is always cast to the type of the variable. For example:

1

2		 char c = 6;

int d = c;

The variable d, which represents an int type, is assigned a value of type char, so the compiler converts the value from char to int.

At the same time, transformations may not always be safe, since different types have different internal representations. And just so moving from one view to another without losing data accuracy is not always possible.

Let's consider what transformations the compiler uses in arithmetic operations:

  1. If one of the operands is of type long double, then the second operand will also be converted to type long double
  2. If the previous item is not executed and if one of the operands is of type double, then the second operand will also be converted to type double
  3. If the previous item is not executed and if one of the operands is of type float, then the second operand will also be converted to type float
  4. If the previous item is not executed and if one of the operands is of type unsigned long int, then the second operand will also be converted to type unsigned long int
  5. If the previous item is not executed and if one of the operands is of type long, then the second operand will also be converted to type long
  6. If the previous item is not executed and if one of the operands is of type unsigned, then the second operand will also be converted to type unsigned
  7. If the previous paragraph is not satisfied, then both operands are converted to int type

Conversion operation

Using a special conversion operation, we can explicitly cast the data to the desired type. For example:

int b = 4;

int c = a / b;

double d = a / b;

double e = (double)a / (double)b;

printf("c = %d \n", c);

printf("d = %f \n", d);

printf("e = %f \n", e);

In the expression int c = a / b; the result of division will be integer - 2, in which the fractional part will be discarded, since both operands of the operation represent integers.

In the expression double d = a / b; the division result will represent a real number - 2.00000 , but since both operands are integers, then again the result of the operation will be an integer 2, and only the division field will assign the result to the variable d, converting the value 2 from int to double.

In the expression double e = (double) a / (double) b, an explicit data conversion is applied to the double type, therefore, the division result will also represent a real number - 2.50000.

To perform the cast operation, the type to which the value must be cast is indicated in parentheses:

4		 int number = 70;

char symbol = (char) number;

printf("symbol = %c \n", symbol);

printf("symbol (int code) = %d \n", symbol);

In some cases, the transformation is accompanied by a loss of information. Without loss of information, the following transformation chains go through:

char -> short -> int -> long

unsigned char -> unsigned short -> unsigned int -> unsigned long

float -> double -> long double

With all other transformations that are not included in these chains, we may encounter a loss of data accuracy. So, in the example above, the conversion from int to char is not safe, therefore, such conversions should be treated with caution.
